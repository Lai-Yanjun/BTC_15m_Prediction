# 树莓派网络代理与连通性一键排障（Predicta Live）

适用场景：
- 你在合规地区，但树莓派需要走 Clash 代理才能访问 Polymarket。
- 你要确保 `gamma-api` 与下单链路都走同一出口。

---

## 0) 进入项目目录

```bash
cd ~/apps/predicta-live
```

---

## 1) 检查 Clash 端口是否在监听

```bash
ss -lntp | rg "7890|7891|7897"
```

常见端口：
- `7890`：HTTP 代理
- `7891/7897`：SOCKS 代理（以你本机配置为准）

---

## 2) 统一代理协议（避免 socks:// 报错）

`httpx` 不接受 `socks://`，必须用 `socks5://`。  
推荐优先使用 HTTP 代理，兼容性最高。

```bash
# 清理历史错误变量
unset ALL_PROXY all_proxy

# 推荐：统一 HTTP 代理
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
export NO_PROXY=localhost,127.0.0.1
export no_proxy=localhost,127.0.0.1

# 如果你必须走 SOCKS，请使用 socks5://（不要用 socks://）
# export ALL_PROXY=socks5://127.0.0.1:7897
# export all_proxy=socks5://127.0.0.1:7897
```

---

## 3) 对比出口 IP（确认代理生效）

```bash
# 直连出口
curl -s https://api.ipify.org ; echo

# 走 HTTP 代理出口
curl -x http://127.0.0.1:7890 -s https://api.ipify.org ; echo
```

两次 IP 应该不同；第二个是代理出口。

---

## 4) 检查 Python 依赖与解释器

```bash
python3 -m venv .venv
./.venv/bin/python -m pip install -U pip
./.venv/bin/python -m pip install -r requirements.txt
./.venv/bin/python -c "import ccxt,requests,httpx; print('deps ok')"
```

注意：运行脚本请优先用 `./.venv/bin/python`，避免系统 Python 缺包。

---

## 5) 验证 gamma-api（使用当前代理环境）

```bash
./.venv/bin/python check_gamma_api.py --config trade_config.yaml
```

成功标准：
- 输出 `ok: true`
- 有 `condition_id`、`token_up`、`token_down`

---

## 6) 单次交易链路冒烟测试（不下单）

```bash
./.venv/bin/python run_live_model.py --run-once --shadow --proxy http://127.0.0.1:7890
```

---

## 7) 启动实盘（前台）

```bash
./.venv/bin/python run_live_model.py --proxy http://127.0.0.1:7890
```

---

## 8) systemd 固化代理（后台常驻必做）

```bash
sudo systemctl edit predicta-live.service
```

写入以下覆盖内容：

```ini
[Service]
Environment="HTTP_PROXY=http://127.0.0.1:7890"
Environment="HTTPS_PROXY=http://127.0.0.1:7890"
Environment="http_proxy=http://127.0.0.1:7890"
Environment="https_proxy=http://127.0.0.1:7890"
Environment="NO_PROXY=localhost,127.0.0.1"
Environment="no_proxy=localhost,127.0.0.1"
Environment="GAMMA_TRUST_ENV_PROXY=true"
Environment="GAMMA_TIMEOUT_SEC=12"
Environment="GAMMA_RETRIES=3"
Environment="GAMMA_BACKOFF_BASE_SEC=0.25"
```

重载并重启：

```bash
sudo systemctl daemon-reload
sudo systemctl restart predicta-live
sudo systemctl status predicta-live
journalctl -u predicta-live -f
```

## 9) 常见报错与处理

### A. `ModuleNotFoundError: No module named 'ccxt'`

原因：用错解释器（系统 Python 而非 `.venv`）。  
处理：按第 4 步安装依赖，并使用 `./.venv/bin/python` 启动。

### B. `Unknown scheme for proxy URL 'socks://...'`

原因：代理协议写成 `socks://`。  
处理：改为 `socks5://`，或直接用 `http://127.0.0.1:7890`。

### C. `AUTO_MARKET_ERROR:ReadTimeout`

原因：边界时刻网络抖动。  
处理：已内置重试；若仍频繁可提高：

```bash
export GAMMA_TIMEOUT_SEC=15
export GAMMA_RETRIES=4
```

### D. `PolyApiException ... 403 ... geoblock`

含义：请求已到 CLOB，但被地区策略拒绝。  
处理：先确认服务进程出口 IP（第 3 步）；若出口不符，调整代理路由与节点。

```bash
cd ~/apps/predicta-live && \

./.venv/bin/python -c "import ccxt,requests,httpx;print('deps ok')" && \

./.venv/bin/python run_live_model.py --run-once --shadow --proxy http://127.0.0.1:7890
```
